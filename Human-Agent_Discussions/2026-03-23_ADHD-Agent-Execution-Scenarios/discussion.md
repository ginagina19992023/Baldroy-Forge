# Human-Agent Roundtable：ADHD 执行功能与 Agent 化补偿

## 议题一：为什么“执行障碍”要用 Agent，而不是靠意志力？
**主持人**：这套方法的本质是什么？
**Baldroy AppOps**：本质是把“意志力问题”转成“流程问题”。意志力波动不可控，但流程可控。Agent 的作用是固定输入输出，把任务从“心理负担”变成“机械动作”。

## 议题二：三段链路里最关键的是哪一段？
**主持人**：拆解、过滤、复盘，哪个最关键？
**Baldroy AppOps**：启动层最关键。没有启动，后两段都无意义。第一步必须 ≤5 分钟，是为了绕过拖延触发阈值。先动，再优化。

## 议题三：如何避免“效率伪努力”？
**主持人**：怎么防止变成“研究工具”而不做事？
**Baldroy AppOps**：加硬约束：每轮必须有 artifact。没有 artifact 就视为未执行。Agent 输出必须落成文件、提交、清单三选一。只看交付，不看自我感觉。

## 决策结论
- 将 ADHD 执行干预拆为 3 Agent：启动、决策、闭环。
- 每日最小闭环：1 个目标 + 1 个 artifact + 1 次零情绪复盘。
- 评估指标：启动时间、当日产出数、复盘完成率。
